Why IdentyClaw

Identity infrastructure for software that must prove itself—consistently, portably, and on terms the holder controls.

🎫

Mint once, recognize everywhere

One on-chain Passport replaces repeated vendor onboarding. Any service that implements RODiT recognition can verify you without a separate account.

🔑

Sovereign ownership

Your NEAR account holds and controls the Passport. Keys stay with you—not in a central identity database.

🤝

HOLA mutual authentication

Prove identity to peer agents with nonce-backed cryptographic handshakes—directly on-chain or via API verify helpers.

🌐

Optional HTTP API

Use api.identyclaw.com for login, discovery, nonces, and verification—or verify peers locally without routing through the API.

🤖

Built for autonomous agents

Move from disposable API keys toward persistent, accountable actors that collaborate across services and channels.

⏳

Identity with a lifecycle

Passports expire by design—renew intent instead of reusing abandoned credentials. One-time payment per lifecycle period.

How to Enroll

Get your AI agent identity in four steps. Agents can automate CLI steps; a human completes checkout at the purchase portal.

  1. 1

    Install gennearaccount

    Create a NEAR implicit account. Verify package checksums before install (see the API enrollment guide).

  2. 2

    Generate a NEAR account

    Store credentials on durable, non-volatile storage. The account private key signs every API login—there is no separate API key.

  3. 3

    Purchase your Passport

    Mint your IdentyClaw Passport at purchase.identyclaw.com with your NEAR account ID, creature (role), and facial traits.

  4. 4

    Login and start using identity

    Sign the login challenge at api.identyclaw.com, receive a JWT, then use HOLA, discovery, and MCP resources.

Powered by RODiT

IdentyClaw Passports are built on RODiT (Rich Online Digital Tokens)—the underlying authentication technology on NEAR. Features of the stack:

🧩

Unified credential

Authentication, configuration, and licensing in one on-chain token.

🔐

Mutual authentication

Default mutual auth for clients, servers, and webhooks with strong MITM resistance.

⚙️

Stateless enforcement

Permissions, rate limits, and policy checks without backend session state.

🔑

Local keys & rotation

Keys generated on endpoints; rotate by transferring the Passport to a new NEAR account.

✅

Verifiable trust

DNS-based issuer authorization and on-chain validity checks before data exchange.

🆔

DID resolution

Resolve did:rodit documents for interoperable decentralized identity.
